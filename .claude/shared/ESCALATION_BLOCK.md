# ESCALATION_BLOCK.md — 에스컬레이션 공통 블록

> agent_runtime/shared/ 의 공통 참조 파일.
> 모든 command와 agent는 자체 에스컬레이션 형식을 만들지 말고 이 파일 형식을 사용한다.
> WORKFLOW.md에는 에스컬레이션 원칙만 요약하고, 실제 형식은 이 파일이 담당한다.

---

## 에스컬레이션 조건

아래 중 하나라도 해당하면 **즉시 멈추고** 사람에게 보고한다.

- 두 선택지가 동등하게 합리적일 때
- CONSTITUTION 불변 원칙과 현실이 충돌할 때
- 작업 범위가 예상의 3배 이상 커질 것 같을 때
- 외부 의존성(패키지, API, DB 스키마) 변경이 필요할 때
- 같은 오류로 3회 이상 재시도 실패했을 때
- `docs/exec/open-decisions.md`의 🔴 Blocker 항목을 만났을 때
- Small Feature로 시작했는데 Large Feature 기준에 해당됨을 감지했을 때

**원칙**: 추측으로 계속 진행하지 않는다. 부분 실행 금지 (파일 수정, 커밋 금지).

---

## 보고 형식

```markdown
## 🚨 에스컬레이션

**상황**: (1~2줄)
**원인 조건**: (위 조건 중 어느 것)
**시도한 것**: (멈추기 전 무엇을 했는지)
**막힌 이유**: (왜 자율 결정 불가)

**선택지**:
- A) [옵션 A] — [장단점]
- B) [옵션 B] — [장단점]
- C) [옵션 C] — (있으면)

**추천**: (선택지 중 하나 + 이유, 없으면 "-")
**영향 범위**: (파일/기능)
```

## 조건 유형별 필수 포함 정보

| 조건 유형 | 추가로 반드시 포함 |
|----------|-----------------|
| 선택지 충돌 | 2~3개 선택지 + 장단점 + 추천 근거 |
| CONSTITUTION 충돌 | 모순되는 원칙 + 현실 상황 (개정 제안은 사람 판단 후에만) |
| 작업 범위 확대 | 현재 진행도, 남은 예상량, 선택지 (분리/계속/롤백) |
| 외부 의존성 필요 | 의존성 목록 + 이유, 대안 검토 결과 |
| 3회 반복 실패 | 시도별 에러 로그, 원인 가설, 수동 개입 지점 |
| Blocker 접촉 | open-decisions 항목 인용, 우회 가능 여부 |
| Small→Large 전환 | 무엇이 바뀌었는지, 현재까지 작업 내용 |

## 사람 응답 패턴

| 응답 | 에이전트 행동 |
|------|------------|
| `A로 진행` (또는 B/C) | 즉시 재개 |
| `잠깐 멈춰` | 대기, 세션 종료 루틴 준비 |
| `방향 바꿔: [새 지시]` | PLANS.md 업데이트 후 새 지시로 재개 |
| `CONSTITUTION 개정 필요` | 개정 완료까지 대기 (실행 금지) |

---

## 응답 없이 세션 종료 시

1. `docs/exec/NEXT_SESSION.md`에 에스컬레이션 상태 명시
2. 변경사항 staged 유지 (가능하면 `git stash`)
3. 다음 세션 시작 시 사람 응답 확인 후 재개

핸드오프 추가 필드:
```markdown
**상태**: ⚠️ 에스컬레이션 대기 중
**원인 조건**: [조건명]
**대기 중인 결정**: [선택지 A / B / C]
**작업 상태**: staged / stashed / 미변경
```
