# 책장 웹서비스 스펙 — bookshelf.md

---

## 개요

Notion "나의 책장" DB를 이관한 데이터를 시각적으로 탐색하는 웹 화면.
운섭이 자신의 독서 기록을 빠르게 찾고, 책별 메모·하이라이트를 다시 읽을 수 있게 한다.

---

## 화면 목록

| 화면 | 경로 | 설명 |
|------|------|------|
| 책 목록 | `/bookshelf` | 갤러리/표 뷰 + 필터 + 검색 |
| 책 상세 | `/bookshelf/[id]` | 메타데이터 + 페이지 내용 |

## 책 목록 화면 (`/bookshelf`)

### 뷰 모드
- **갤러리 뷰**: 표지 이미지 중심 카드 그리드. 카드에 표시: 제목, 저자, 별점, 상태 배지
- **표 뷰**: 전체 컬럼 (제목, 저자, 분류, 상태, 별점, 읽은 기간, 한단어감상, 한줄 요약)
- 우측 상단 토글 버튼으로 전환

### 필터 (복합 적용 가능)
| 필터 | 타입 | 옵션 |
|------|------|------|
| 상태 | 단일 선택 | 완독, 읽는 중, 읽고 싶은 책, 완독 2회차, 중단, 소장, 서평완료, 속독, 상시 |
| 분류 | 단일 선택 | 에세이, 자기계발, 경제경영, 사회과학, 인문, 소설류 등 34개 |
| 별점 | 단일 선택 | ★ ~ ★★★★★ |
| 키워드 | 멀티 선택 | DB의 키워드 목록 전체 |

### 검색
- 텍스트 검색: 제목, 저자, 한줄 요약, 한 줄 평 대상
- 실시간 검색 (디바운스 300ms)

### 정렬
- 기본: 읽은 기간 내림차순 (최근 읽은 것 먼저)
- 선택: 별점 높은 순, 등록일 순, 제목 가나다순

### 통계 헤더 (상단)
- 총 책 수, 완독 수, 이번 달 완독 수

## 책 상세 화면 (`/bookshelf/[id]`)

### 메타데이터 섹션
- 표지 이미지 (좌측)
- 제목, 저자
- 분류, 상태, 별점 (★ 표시)
- 읽은 기간
- 키워드 배지 목록
- 한단어감상 배지
- 한줄 요약
- 한 줄 평

### 페이지 내용 섹션
책 상세 페이지(Notion)에서 이관된 텍스트를 content_type 별로 구분해 표시.

| content_type | 표시 방식 | 설명 |
|-------------|---------|------|
| `highlight` | 노란 배경 인용 블록 | 책에서 발췌한 문장 |
| `memo` | 일반 텍스트 + 페이지 번호 | 개인 생각·코멘트 |
| `ai_chat` | 회색 배경 AI 대화 블록 | Gemini/Claude와의 독서 토론 |
| `diary` | 초록 배경 일기 블록 | 책과 연결된 개인 일기·회고 |

탭 또는 토글로 content_type 필터링 가능.

> **메모 추가**: 책 상세 화면에서 직접 메모를 작성하는 기능은 Phase 4 대상.
> Phase 2에서는 읽기 전용으로 이관된 데이터를 표시한다.
> Phase 4 구현 시 API 엔드포인트: `POST /api/books/[id]/pages` (book_pages 테이블 insert)

### 네비게이션
- 상단: ← 책장으로 돌아가기
- 하단: 이전 책 / 다음 책 (읽은 기간 순)

## Notion 데이터 → DB 매핑

| Notion 필드 | DB 컬럼 | 비고 |
|------------|---------|------|
| 이름 | books.title | |
| 저자 | books.author | |
| 분류 | books.category | |
| 상태 | books.status | |
| 별점 | books.rating | ★=1, ★★★★★=5 변환 |
| 키워드 | books.keywords | 배열 |
| 한단어감상 | books.one_word | 배열 |
| 한줄 요약 | books.summary | |
| 한 줄 평 | books.review | |
| 표지 | books.cover_url | URL 추출 |
| 읽은 기간 | books.read_start, read_end | |
| AI 독서메이트 (URL) | books.notion_ai_url | Gemini 링크 보존 |
| 페이지 내용 (블록) | book_pages 테이블 | content_type 자동 분류 |

### 페이지 내용 content_type 자동 분류 규칙

Notion 블록을 파싱할 때 아래 순서로 판단한다.

| 우선순위 | 조건 | content_type |
|---------|------|-------------|
| 1 | 노란색 하이라이트(`yellow_bg`) 블록 | `highlight` |
| 2 | `숫자p` 패턴으로 시작 (예: `225p`, `47p`) | `memo` |
| 3 | 날짜(YYYY.MM.DD) 패턴으로 시작하는 200자 이상 블록 | `diary` |
| 4 | "블루", "Gemini", "Claude" 등 AI 응답 패턴 | `ai_chat` |
| 5 | 그 외 | `memo` |

> **주의**: 실제 Notion 데이터의 페이지 번호 표기는 `숫자p` 형식 (예: `225p`)이다. `p숫자` 형식이 아님.
