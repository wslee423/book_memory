# AI 독서메이트 스펙 — ai-mate.md

> Phase 3 구현 대상. Phase 2 완료 후 착수.

---

## 개요

내 독서 기록(하이라이트·메모·AI 대화·일기)에서 자연어로 관련 내용을 찾아주는 RAG 기반 개인 비서.
단순 검색이 아니라, 내 생각과 메모를 다시 꺼내 현재 상황·질문에 연결해 답변한다.

---

## 핵심 사용 시나리오

| 시나리오 | 예시 질문 |
|---------|---------|
| 특정 주제 회상 | "내가 불편함을 피하려고만 하면 생기는 문제에 대해 읽었던 내용 찾아줘" |
| 자료 정리 요청 | "자산관리 세미나에 쓸 만한 독서 메모 정리해줘" |
| 패턴 분석 | "최근 별점 5개 준 책들의 공통점을 분석해줘" |
| 생각 연결 | "요즘 육아 때문에 고민인데, 읽은 책 중에 관련 내용 있어?" |
| 인사이트 추출 | "투자에 관해 내가 남긴 메모들 중 핵심만 뽑아줘" |

## RAG 파이프라인 상세

### 임베딩 단위 (청킹 전략)
- book_pages 테이블의 각 row를 임베딩 단위로 사용
- 단, content 길이가 500자 초과 시 500자 단위로 분할
- 책 메타데이터(제목+저자+키워드+한줄요약)도 별도 임베딩 생성 → source_type: 'book_meta'

### 검색 전략
```
1. 질문 임베딩 생성
2. pgvector cosine 유사도 검색 (top-10 후보)
3. 책 메타데이터 + 페이지 내용 혼합 검색
4. 중복 book_id 제거 후 top-5 청크 선택
5. 각 청크에 원본 책 메타데이터(제목, 저자, 날짜) 붙여서 컨텍스트 구성
```

### 시스템 프롬프트 핵심 지침
```
당신은 운섭의 개인 AI 독서 비서입니다.
운섭이 읽은 책에서 남긴 메모, 하이라이트, 생각들을 기반으로 답변하세요.
반드시 제공된 독서 기록에서 근거를 찾아 답변하고,
어떤 책의 어떤 내용을 참조했는지 출처를 명시하세요.
독서 기록에 없는 내용은 "관련 독서 기록을 찾지 못했습니다"라고 답하세요.
```

### 응답 형식
- 답변 본문 (자연어)
- 참조 출처 카드: 책명 + 저자 + 발췌 내용 미리보기

## 채팅 UI (`/mate`)

### 레이아웃
- 좌측: 대화 이력 목록 (세션별)
- 우측: 현재 대화 창

### 대화 창 구성
- 상단: 세션 제목 (첫 질문 기반 자동 생성)
- 중앙: 메시지 스트림 (스트리밍 응답 지원)
- 하단: 입력창 + 전송 버튼

### 메시지 타입
| 타입 | 표시 |
|------|------|
| user | 우측 정렬, 파란 배경 |
| assistant | 좌측 정렬, 회색 배경 + 참조 출처 카드 |

### 참조 출처 카드
```
📖 [책 제목] — [저자]
"[발췌 텍스트 미리보기 100자]..."
```
카드 클릭 시 해당 책 상세 페이지로 이동.
