# CONSTITUTION.md — book_memory 헌법

> 이 문서는 모든 문서 중 최상위 권위를 가진다.
> 에이전트와 사람 모두 이 원칙에 따라 행동한다.
> **개정 실행은 사람만** 할 수 있다.

---

## 1. 프로젝트 목적

읽은 책과 함께 남긴 생각·메모·하이라이트를 AI가 다시 꺼내 쓸 수 있게 해주는 **개인 AI 독서 비서 서비스**다.
단순한 책 보관함이 아니라, 독서 기록을 삶의 의사결정에 연결해주는 도구를 만든다.
사용자는 운섭 1인이며, 외부 공개 없이 순수 개인 도구로 운영한다.

---

## 2. 문서 위계

충돌 시 상위 문서가 이긴다.

```
CONSTITUTION.md              ← 최상위
  ↓
CLAUDE.md                    ← 에이전트 운영 규칙
  ↓
AGENTS.md                    ← 역할 및 자율 범위
  ↓
QUALITY_SCORE.md             ← 품질 기준
  ↓
ARCHITECTURE.md              ← 기술 구현 기준 (스펙보다 상위)
  ↓
docs/product-specs/*.md      ← 기능 스펙
```

하위 문서가 상위 문서와 모순되면, 에이전트는 상위 문서를 따르고 사람에게 보고한다.

---

## 3. 에이전트 자율 범위

`AGENTS.md`의 **"자율 결정 가능한 것"** 으로 명시된 범위 내에서만 사람 승인 없이 결정한다.
**명시되지 않은 것은 자율 범위가 아니다.**

---

## 4. 충돌 해결

### 4-1. 에이전트 간 충돌
QA 성격 에이전트는 `QUALITY_SCORE.md` 미달을 이유로 **거부권** 행사 가능.
그 외 충돌은 사람이 최종 결정.

### 4-2. 에이전트 ↔ 문서 충돌
임의 결정 금지. 사람에게 보고 후 지시받는다.

### 4-3. 문서 간 충돌
섹션 2 위계 따름. 위계로 해결 안 되면 사람에게 보고.

---

## 5. 불변 원칙 (Immutable Principles)

어떤 상황에서도, 어떤 에이전트도 어길 수 없다.

### 원칙 1 — 데이터 손실 절대 금지
Notion 원본 데이터 및 Supabase 데이터는 어떤 작업에서도 영구 삭제하지 않는다.
검증: 마이그레이션·배치 작업 전 반드시 백업 확인 단계가 있어야 한다.

### 원칙 2 — 개인 메모는 AI 학습에 사용하지 않는다
하이라이트·일기·개인 생각은 LLM API 호출 시 일회성 컨텍스트로만 사용한다.
검증: 외부 학습 파이프라인 연결 코드 발견 즉시 차단.

### 원칙 3 — 외부 공개 엔드포인트 없음
인증 없이 접근 가능한 API Route는 존재할 수 없다.
검증: 모든 API Route에 인증 미들웨어 적용 여부 확인.

### 원칙 4 — UI 텍스트는 한국어
모든 화면의 UI 텍스트(버튼, 레이블, 메시지, 에러)는 한국어다.
검증: 영문 UI 텍스트 발견 즉시 FAIL.

### 원칙 5 — Notion은 읽기 전용 백업
마이그레이션 완료 후 Notion DB는 읽기 전용 백업으로만 유지한다.
Supabase가 단일 진실 소스(Source of Truth)다.
검증: 앱이 Notion API에 데이터를 쓰는 코드가 없어야 한다.
